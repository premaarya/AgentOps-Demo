STATEMENT OF WORK #001 - CLOUD MIGRATION

This Statement of Work ("SOW") is entered into as of March 1, 2026, under the Master Service Agreement dated February 1, 2026, between GlobalTech Solutions LLC ("Provider") and Meridian Enterprises Inc. ("Client").

1. PROJECT OVERVIEW
Provider shall migrate Client's on-premises infrastructure to Microsoft Azure cloud platform, including application modernization, data migration, and security hardening.

2. SCOPE OF WORK
2.1 Phase 1 - Assessment (Weeks 1-2): Infrastructure audit, application dependency mapping, migration readiness assessment, and risk analysis.
2.2 Phase 2 - Design (Weeks 3-4): Cloud architecture design, network topology, security framework, and disaster recovery planning.
2.3 Phase 3 - Migration (Weeks 5-10): Staged migration of applications, databases, and storage to Azure with minimal downtime.
2.4 Phase 4 - Validation (Weeks 11-12): Performance testing, security validation, compliance verification, and go-live support.

3. DELIVERABLES
3.1 Cloud Readiness Assessment Report
3.2 Azure Architecture Design Document
3.3 Migration Runbook with rollback procedures
3.4 Migrated infrastructure on Azure (12 VMs, 3 databases, 2 storage accounts)
3.5 Security Compliance Report
3.6 Knowledge Transfer documentation and training sessions (3 sessions, 2 hours each)

4. TIMELINE
Project duration: 12 weeks from project kickoff.
Kickoff date: March 15, 2026.
Target completion: June 7, 2026.

5. FEES
5.1 Total project fee: $285,000 (Two Hundred Eighty-Five Thousand Dollars).
5.2 Payment schedule:
    - 25% upon SOW execution ($71,250)
    - 25% upon completion of Phase 2 ($71,250)
    - 40% upon completion of Phase 3 ($114,000)
    - 10% upon final acceptance ($28,500)
5.3 Travel expenses billed at cost with prior approval, not to exceed $15,000.

6. ACCEPTANCE CRITERIA
6.1 All applications functional on Azure with response times within 10% of baseline.
6.2 Zero data loss during migration.
6.3 Security controls meet or exceed current on-premises standards.
6.4 Client acceptance testing period: 10 business days from delivery.

7. ASSUMPTIONS AND DEPENDENCIES
7.1 Client provides VPN access to on-premises environment within 5 business days of kickoff.
7.2 Client assigns a dedicated project manager and technical lead.
7.3 Azure subscription provisioned by Client prior to Phase 3.

8. CHANGE MANAGEMENT
Any changes to scope, timeline, or deliverables must be documented in a Change Order signed by both parties. Change orders may adjust fees and timeline.

9. WARRANTY
Provider warrants that all deliverables will conform to the specifications in this SOW for a period of ninety (90) days following final acceptance.
